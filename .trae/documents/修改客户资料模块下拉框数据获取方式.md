# 修改客户资料模块下拉框数据获取方式

## 目标
将 base_cust 目录下的下拉框数据获取方式改为与 base_part/addInfor.vue 中相同的方式，即从 API 获取数据并动态生成下拉选项。

## 分析
1. **base_part/addInfor.vue** 中的下拉框数据获取方式：
   - 在组件挂载时调用 `fetchPartInfoData()` 函数
   - 通过 `apiClient.get('/part-info')` 获取配件信息列表
   - 从配件信息中提取唯一的车型、单位、品牌、规格值
   - 转换为下拉框选项格式并赋值给对应的 options 变量

2. **base_cust/index.vue** 中的问题：
   - 客户类型和客户地区下拉框的 `:options` 属性为空
   - 没有实现从 API 获取下拉框数据的功能

3. **base_cust/addInfor.vue** 中的问题：
   - 所属银行、客户类型、经营范围、所属地区、付款方式、默认物流、默认运货方式的选项都是硬编码的静态数据
   - 没有实现从 API 获取下拉框数据的功能

## 实现计划

### 1. 修改 base_cust/addInfor.vue
- 添加 `fetchCustomerOptions()` 函数，从 API 获取客户相关的下拉框数据
- 移除硬编码的静态选项数据
- 在组件挂载时调用 `fetchCustomerOptions()` 函数
- 为每个下拉框创建对应的 ref 变量存储选项数据

### 2. 修改 base_cust/index.vue
- 添加 `fetchCustomerOptions()` 函数，从 API 获取客户相关的下拉框数据
- 为客户类型和客户地区下拉框创建对应的 ref 变量存储选项数据
- 在组件挂载时调用 `fetchCustomerOptions()` 函数
- 将下拉框的 `:options` 属性绑定到对应的 ref 变量

## 预期结果
- 客户资料模块的所有下拉框数据都从 API 获取
- 下拉框选项会根据 API 返回的数据动态生成
- 实现方式与配件资料模块的下拉框数据获取方式一致
