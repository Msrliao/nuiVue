# 实现员工资料页面

## 分析现有页面结构

通过分析现有三个页面（客户资料、配件资料、仓库资料），发现它们都遵循以下共同模式：

1. **目录结构**：每个模块都有独立目录，包含 `index.vue`、`table.vue` 和 `addInfor.vue` 组件
2. **技术栈**：Vue 3 Composition API + TypeScript + Naive UI + apiClient
3. **功能模块**：
   - 搜索表单（带防抖功能）
   - 数据表格展示
   - 新增/编辑功能
   - 刷新数据功能

## 员工资料页面设计

### 1. 创建目录结构

```
src/views/base/base_emp/
├── index.vue          # 主页面
├── table.vue          # 员工数据表格
└── addInfor.vue       # 员工信息添加/编辑组件
```

### 2. 实现 index.vue 主页面

- 包含员工信息搜索表单（工号、姓名、部门、职位）
- 新增员工按钮
- 员工数据表格
- 员工信息编辑模态框
- 防抖搜索功能
- API 调用获取员工数据

### 3. 实现 table.vue 表格组件

- 展示员工列表数据
- 包含编辑操作按钮
- 支持分页和排序

### 4. 实现 addInfor.vue 添加/编辑组件

- 员工信息表单（工号、姓名、性别、出生日期、联系电话、邮箱、职位、部门、入职日期、状态）
- 保存和取消按钮
- 支持编辑现有员工信息

### 5. API 接口设计

- 参考现有页面的 API 调用模式，使用 `/employees` 接口
- 支持 GET（获取列表）、POST（新增）、PUT（更新）操作

## 实现步骤

1. 创建 `base_emp` 目录结构
2. 实现 `table.vue` 组件
3. 实现 `addInfor.vue` 组件
4. 实现 `index.vue` 主页面
5. 测试页面功能

## 技术要点

- 遵循现有代码风格和命名规范
- 使用相同的组件库和工具函数
- 保持与现有页面一致的用户体验
- 确保代码可维护性和可扩展性
